# 001-結合セルの罫線が適切に表示されない

## 発生日
2026-03-06

## 対象レイヤー
domain / web

## 概要

結合セルの罫線が正しく表示されないケースがある。`collectMergeBorder` による外周セル収集ロジックを実装済みだが、実際の Excel ファイルでは改善が見られない。

## 発生状況

印刷領域の整合ロジック実装後、実際の .xlsx ファイルでプレビューを確認したところ、結合セルの罫線（特に右辺・下辺）が欠落するケースを確認。

## 調査経緯

- 2026-03-06: 初期調査 — マスター（左上）セルの罫線のみ使用していたのが原因と判断し、`collectMergeBorder` を実装。ユニットテストは通過するが、実ファイルでは改善せず。
- 2026-03-06: 追加仮説 — 以下の可能性がある:
  1. **Web 層の parseExcelFile でスレーブセルが渡されていない**: `eachCell({ includeEmpty: false })` によりスタイルのみ・値なしのスレーブセルがスキップされている可能性
  2. **ExcelJS のスレーブセル罫線取得の挙動**: PoC では罫線が取得できていたが、実ファイルでは異なるパターンがあり得る
  3. **CSS レンダリング側の問題**: ボーダー情報は正しく Box に格納されているが、CSS 描画で欠落している可能性

## 関連ドキュメント

- `src/domain/excel/BorderConverter.ts` — 罫線変換ロジック
- `src/web/services/parseExcelFile.ts` — Excel ファイルのパース（Web 層）
- `poc/exceljs/REPORT.md` — ExcelJS PoC（深掘りA: 結合セルの罫線スタイル）
