# 007-非表示行・列の対応

## 状態
未着手

## 担当
（未定）

## 対象パッケージ
web / domain

## 概要

XLSX の非表示行（`<row hidden="1">`）・非表示列（`<col hidden="1">`）を検出し、ボックス生成時に除外またはゼロサイズとして扱う。

## 背景

非表示行/列を含む帳票をアップロードすると、非表示セルもボックスとして生成されレイアウトが崩れる。特に計算用の隠しセルを持つ帳票で問題になる。

## 前提条件チェックリスト
- [x] 軽量 XLSX パーサーが動作していること

## 実装チェックリスト
- [ ] `SheetParser.ts`: `<row hidden="1">` と `<col hidden="1">` を検出
- [ ] 方式選択: 高さ/幅を 0 にする or `RawSheetData` に `hiddenRows`/`hiddenCols` を追加
- [ ] `ExcelParser.ts`: 非表示行/列のセルをボックス生成から除外
- [ ] テスト作成

## 備考

方式 A（高さ/幅を 0）はシンプルだが座標計算に影響する可能性あり。方式 B（除外フラグ）は ExcelParser 側の変更が必要だが確実。
