# 未実装項目管理（バックログ）

実装予定だが未着手の項目を管理する。

## 目的

- 未実装項目の可視化と優先順位の管理
- 前提条件の充足状況の追跡
- 担当の明確化

## 運用ルール

### 4状態

| 状態 | 意味 |
|------|------|
| 未着手 | 前提条件が未充足、または優先度が低い |
| 着手可能 | 前提条件が充足し、着手可能な状態 |
| 進行中 | 実装中 |
| 完了 | 実装完了・レビュー済み |

### ファイル命名

```
docs/backlog/items/{連番3桁}-{kebab-case}.md
```

例: `001-excel-parser-print-area.md`

### テンプレート

```markdown
# {連番}-{タイトル}

## 状態
未着手 / 着手可能 / 進行中 / 完了

## 担当
（担当者名またはエージェント名）

## 対象パッケージ
domain / web

## 概要
（何を実装するか）

## 背景
（なぜ必要か）

## 前提条件チェックリスト
- [ ] 条件1
- [ ] 条件2

## 実装チェックリスト
- [ ] 実装項目1
- [ ] 実装項目2
- [ ] テスト作成

## 備考
（補足事項）

## 関連ドキュメント
（関連する制約条件やADR）
```

### 段階分離パターン

大きな項目は以下のように分割する:

- **Step A**: 外部サービスなしで可能な範囲（ローカル動作）
- **Step B**: 外部サービス必須の範囲
