# 001-ボックスモデル座標系

## 発生日
2026-03-05

## 対象レイヤー
domain

## 概要
ボックスモデルの座標系はすべて mm（ミリメートル）単位の絶対座標とする。

## 背景
Excel帳票をCSS帳票に変換するにあたり、統一した座標系が必要。Excelの列幅は「文字数単位」、行高は「ポイント単位」で定義されており、CSS での配置には物理単位（mm）が最適。印刷時の精度を保証するためにも mm を採用する。

## 詳細
- ボックスの位置・サイズ: `x`, `y`, `w`, `h` すべて mm 単位
- 線分の座標: `x1`, `y1`, `x2`, `y2` すべて mm 単位
- Excel の列幅（文字数単位）→ mm への変換は `paper/CoordinateConverter` が担当
- Excel の行高（pt 単位）→ mm への変換: `1pt = 0.3528mm`
- CSS 出力時: `position: absolute` + mm 単位で配置

## 影響範囲
- `src/domain/box/` — ボックスの型定義・操作
- `src/domain/line/` — 線分の型定義・操作
- `src/domain/paper/` — 座標変換ロジック
- `src/web/` — SVG プレビュー・エディタの座標マッピング

## 今後の対応
この制約は設計の根幹であり、解消予定なし。変更する場合は全レイヤーへの影響を評価する必要がある。

## 状態
有効
