# 007-テーマカラー / インデックスカラー未対応

## 発生日
2026-03-07

## 対象レイヤー
web

## 概要

軽量 XLSX パーサーの `StylesParser.extractColor()` は RGB 直指定（`<color rgb="AARRGGBB">`）のみ対応。テーマカラー（`<color theme="N">`）とインデックスカラー（`<color indexed="N">`）は解決されず、色情報が欠落する。

## 背景

ExcelJS → 軽量パーサー移行時に「80%ルール」として意図的にスコープ外とした（`StylesParser.ts` 187行目のコメント参照）。しかし、実際の帳票ファイルを分析した結果、テーマカラーの使用率が想定より高く、特にデフォルトの黒文字がテーマカラー `theme="1"` で指定されているケースが多いことが判明。

## 詳細

### 影響を受ける色の種類

| 色の指定方式 | 対応状況 | 影響 |
|-------------|---------|------|
| `<color rgb="AARRGGBB">` | ✅ 対応 | — |
| `<color theme="N">` | ❌ 未対応 | フォント色・罫線色・背景色が欠落 |
| `<color theme="N" tint="0.5">` | ❌ 未対応 | 同上（明度調整も未対応） |
| `<color indexed="N">` | ❌ 未対応 | レガシー Excel ファイルで色が欠落 |

### 実際の影響パターン

1. **フォント色欠落**: テーマカラーで指定された文字色が `undefined` → デフォルト黒にフォールバック（影響軽微）
2. **罫線色欠落**: テーマカラーで指定された罫線色が `undefined` → 罫線は表示されるが色が不正確
3. **背景色欠落**: テーマカラーで指定された塗りつぶし色が `undefined` → 背景色が反映されない

## 影響範囲

- `src/web/services/xlsx/StylesParser.ts` — `extractColor()` 関数
- フォント色、罫線色、背景色の全て

## 解消

バックログ 005 として実装完了（2026-03-07）。

- `ThemeParser.ts`: テーマカラーパレット抽出 + `applyTint()` による明度調整
- `IndexedColors.ts`: OOXML 標準 64 色テーブル
- `StylesParser.ts`: `extractColor()` を拡張し `theme` + `tint` + `indexed` を解決

## 状態
解消（2026-03-07）
